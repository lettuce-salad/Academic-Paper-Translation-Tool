# 最終改善ガイド

## 🎯 実施した改善

### 1. **段落マーカーの改善** ✅

**修正前**:
```
### 段落1 ###
```
→ LLMが削除・変更しやすい

**修正後**:
```
【段落1】
```
→ より明確で保持されやすい

---

### 2. **デフォルトセクションサイズを拡大** ✅

**修正前**: 3000文字
**修正後**: 8000文字

**効果**:
- より広い文脈で翻訳
- 文が途切れにくい
- 自然な日本語

---

### 3. **デバッグ情報の追加** ✅

翻訳中に詳細な情報を表示:

```
[セクション 1/20] 15ブロック
   翻訳結果サンプル: 【段落1】本研究の主要な目標...
   抽出された段落数: 15/15
   成功: 15/15ブロック
```

---

### 4. **より詳細なプロンプト** ✅

```
【重要な指示】
1. 各段落は「【段落N】」という形式で始まります
2. 翻訳後も必ず同じ形式を保持してください
3. 段落番号を変えたり、削除したりしないでください
4. 段落の順序を変えないでください
5. 代名詞は文脈から適切な名詞に置き換えてください
```

---

## 🚀 使い方

### 基本的な使い方（推奨）

```cmd
python main.py paper.pdf --translate
```

**デフォルト設定**:
- セクションサイズ: 8000文字（自動）
- 見出しで自動分割
- 文脈を広く保持

---

### より大きな文脈

```cmd
python main.py paper.pdf --translate --section-size 12000
```

**用途**: 非常に短い論文（5ページ以下）

---

### 標準設定

```cmd
python main.py paper.pdf --translate --section-size 8000
```

**用途**: ほとんどの論文（デフォルト）

---

### より細かく分割

```cmd
python main.py paper.pdf --translate --section-size 5000
```

**用途**: 非常に長い論文（50ページ以上）

## 📊 品質チェック

翻訳中のログを確認:

### 良い例

```
[セクション 5/15] 25ブロック
   翻訳結果サンプル: 【段落1】我々は新しい手法を提案する。【段落2】この手法は...
   抽出された段落数: 25/25  ← 全て抽出できている ✅
   成功: 25/25ブロック       ← 全て成功 ✅
```

---

### 問題がある例

```
[セクション 8/15] 30ブロック
   翻訳結果サンプル: 本研究では新しい...（段落番号なし）
   抽出された段落数: 15/30  ← 半分しか抽出できていない ❌
   成功: 15/30ブロック       ← 失敗が多い ❌
```

**対処法**:
1. セクションサイズを小さくする
2. 別の翻訳エンジンを試す（DeepL推奨）

---

## ✅ 確認ポイント

翻訳後のHTMLで確認:

### 1. 文の完結性

❌ **悪い例**:
```
本研究の主要な目標の一つは、Cachet自体の性能向上にとどまらず、
```
→ 文が途中で切れている

✅ **良い例**:
```
本研究の主要な目標の一つは、Cachet自体の性能向上にとどまらず、
SATソルバーのヒューリスティックを特定することである。
```
→ 文が完結している

---

### 2. 段落の順序

❌ **悪い例**:
```
段落3の内容
段落1の内容
段落2の内容
```
→ 順序が入れ替わっている

✅ **良い例**:
```
段落1の内容
段落2の内容
段落3の内容
```
→ 正しい順序

---

### 3. 代名詞の処理

❌ **悪い例**:
```
深層学習はAIに革命をもたらした。
それは機械がデータから学習することを可能にする。
```
→ "それ" が不明確

✅ **良い例**:
```
深層学習はAIに革命をもたらした。
深層学習により、機械がデータから学習することが可能になる。
```
→ 明示的に記述

---

### 4. 段落間のつながり

❌ **悪い例**:
```
実験を行った。
結果を示す。
```
→ つながりが不自然

✅ **良い例**:
```
実験を行った。
以下、その結果を示す。
```
→ 自然なつながり

## 🔧 トラブルシューティング

### 問題1: 段落番号が抽出できない

**ログ**:
```
抽出された段落数: 5/20  ← 少ない
```

**原因**: LLMが段落番号を削除している

**解決方法**:

#### A. DeepLを使用（最も効果的）

```cmd
# .envファイルを設定
DEEPL_API_KEY=your_api_key

# 翻訳実行
python main.py paper.pdf --translate
```

DeepLは段落番号を確実に保持します。

---

#### B. セクションサイズを小さくする

```cmd
python main.py paper.pdf --translate --section-size 5000
```

小さいセクション = LLMが処理しやすい

---

### 問題2: 文が途中で切れる

**原因**: セクションが大きすぎる

**解決方法**:

```cmd
python main.py paper.pdf --translate --section-size 6000
```

---

### 問題3: 代名詞が不自然

**原因**: セクションが小さすぎて文脈不足

**解決方法**:

```cmd
python main.py paper.pdf --translate --section-size 10000
```
