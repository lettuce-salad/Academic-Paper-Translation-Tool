# ページモード翻訳ガイド

## 🎯 ページモードとは

**ページ全体を一括翻訳**する新しいモードです。

---

## 📊 比較

### セクションモード（デフォルト）

```
1ページ → 複数のセクションに分割 → それぞれ翻訳
```

**特徴**:
- セクション数: 15-30個
- 1セクション: 8000文字
- 翻訳速度: 速い
- 品質: 良い

---

### ページモード（NEW!）

```
1ページ → まとめて翻訳
```

**特徴**:
- ページ数: 論文のページ数
- 1ページ: 全ブロック
- 翻訳速度: やや遅い
- 品質: 最高

---

## 🚀 使い方

### ページモードで翻訳

```cmd
python main.py paper.pdf --translate --page-mode
```

**出力**:
```
📖 PDFを解析中: paper.pdf
   ページ数: 10
   総ブロック数: 245
   翻訳対象: 180ブロック

🔄 翻訳中（ページモード）...

📄 ページ 1/10 を翻訳中...
   🔒 保護: 15個
   ✅ DeepL使用
   🔓 復元完了
   ✅ 45ブロック翻訳完了

📄 ページ 2/10 を翻訳中...
   ...
```

---

## ✅ メリット

### 1. 最も広い文脈

```
ページ全体 → 全ての段落を考慮して翻訳
```

**効果**:
- ✅ 代名詞が最も自然
- ✅ 段落間のつながりが完璧
- ✅ ページ全体の流れが滑らか

---

### 2. 段落の順序が保持

```
【1】段落1
【2】段落2
【3】段落3
...
【45】段落45
```

**効果**:
- ✅ 順序が入れ替わらない
- ✅ 文が途中で切れない

---

### 3. シンプルな処理

```
10ページ → 10回の翻訳
```

**効果**:
- ✅ デバッグしやすい
- ✅ エラーが少ない

---

## ⚠️ デメリット

### 1. 翻訳時間が増加

```
セクションモード: 20分
ページモード: 30-40分
```

**理由**: 1回あたりの翻訳量が多い

---

### 2. ページが大きいと失敗しやすい

```
1ページ = 100ブロック → OK
1ページ = 200ブロック → タイムアウトの可能性
```

**対処**: DeepL使用（推奨）

---

## 📝 推奨設定

### 標準的な論文（1ページ=2カラム）

```cmd
python main.py paper.pdf --translate --page-mode
```

**ページあたり**: 30-50ブロック
**翻訳時間**: 30分程度

---

### 密度の高い論文（1ページに多くのテキスト）

```cmd
# セクションモードを推奨
python main.py paper.pdf --translate
```

**理由**: ページモードだと1回の翻訳量が多すぎる

---

## 🎯 どちらを選ぶ？

### ページモードを推奨

以下の場合:
- ✅ DeepL APIを使用している
- ✅ 最高品質の翻訳が必要
- ✅ 時間に余裕がある
- ✅ 論文のページあたりのテキスト量が標準的

```cmd
python main.py paper.pdf --translate --page-mode
```

---

### セクションモードを推奨

以下の場合:
- ✅ 翻訳速度を優先
- ✅ ページあたりのテキスト量が多い（密度が高い）
- ✅ Ollama（ローカルLLM）を使用
- ✅ とりあえず翻訳したい

```cmd
python main.py paper.pdf --translate
```
